# akashpuri-07.github.io

Static personal site for Akash Puri. No backend, no database — the whole
site is plain HTML/CSS/JS served directly by GitHub Pages.

## How the Journal section works

- **`posts/posts.json`** is the single source of truth for every blog post
  (title, date, excerpt, tags, image, and the full body in Markdown).
- The homepage (`index.html`) fetches `posts/posts.json` in the browser at
  page load and renders the Journal grid from it — no server call, no DB.
- **`blog/{slug}/index.html`** are pre-rendered static pages (one per
  published post) so each article has a clean, crawlable URL and works
  even with JavaScript disabled. They're generated from `posts/posts.json`
  by `scripts/build_blog.py`.
- **`sitemap.xml`** is regenerated by the same script.

## Adding / editing a post

1. Edit `posts/posts.json` — add a new object to the array (or edit an
   existing one). Set `"published": true` when it's ready to go live.
2. Regenerate the static blog pages + sitemap:
   ```bash
   pip install -r scripts/requirements.txt
   python3 scripts/build_blog.py
   ```
3. Commit and push `posts/posts.json`, the updated `blog/` folder, and
   `sitemap.xml`.

Removing a post: set `"published": false` (or delete its entry) and rerun
the build script — its `/blog/{slug}/` folder is removed automatically.

## What changed from the previous version

- Removed the FastAPI + MongoDB backend, Google-OAuth admin dashboard, and
  the `frontend/` CRA scaffold (it was unused boilerplate — the real site
  is the root-level `index.html`).
- The Journal section now reads `posts/posts.json` instead of calling
  `/api/posts`.
- The contact form still POSTs to `/api/messages`, left as-is per request —
  it has no backend to receive it right now.
